[HEADING=2]Installation[/HEADING]
[LIST=1]
[*]Download [URL='https://github.com/MCTheryn/item-magent/releases/latest']ItemMagnet-1.6.6.jar[/URL]
[*]Place the JAR in your server's [CODE]plugins/[/CODE] folder
[*]Restart the server (Paper 1.21.1 or newer, Java 21+)
[*]On first run, ItemMagnet creates [CODE]plugins/ItemMagnet/config.yml[/CODE] and [CODE]messages.yml[/CODE]
[*]Give yourself a magnet: [CODE]/itemmagnet give YourName fragment 500[/CODE]
[*]Tune settings via [CODE]/itemmagnet config[/CODE] or edit [CODE]config.yml[/CODE], then [CODE]/itemmagnet reload[/CODE]
[/LIST]

[HEADING=2]Quick start for players[/HEADING]
[LIST]
[*]Hold a magnet in your [B]main hand[/B] (unless hold-mode is changed in config)
[*]Drop items nearby — they pull toward you with visible motion (line-of-sight physics)
[*]Put redstone dust or a block in either hand; [B]sneak + right-click[/B] to transfer fuel
[*]Redstone blocks add extra charge and a temporary radius boost (power surge)
[*]Use [CODE]/itemmagnet filter[/CODE] to set personal item blacklist rules
[/LIST]

[HEADING=2]Configuration overview[/HEADING]

[HEADING=3]settings[/HEADING]
Core magnet behaviour: scan interval, pull step size, hold mode, vertical reach, pickup distance, item blacklist/tags, XP pull, sounds.

[HEADING=3]tiers[/HEADING]
Define Fragment, Survey, Anchor (or custom tiers): material, display name, lore, radius, drain, recipes, unlock gates, tier filters.

[HEADING=3]fuel[/HEADING]
Redstone dust and block charge values, boost duration, radius bonus per boost level.

[HEADING=3]integrations[/HEADING]
Optional hooks for Lands, WorldGuard, Towny, GriefPrevention, Residence, PlotSquared, SuperiorSkyblock2, CMI, Quests. Each integration is independently enabled.

[HEADING=3]player-filter[/HEADING]
Default preset for new players; storage backend [CODE]YAML[/CODE], [CODE]SQLITE[/CODE], or [CODE]MYSQL[/CODE].

[HEADING=3]metrics[/HEADING]
[LIST]
[*][CODE]update-check[/CODE] — [CODE]ON_STARTUP[/CODE], [CODE]ALWAYS[/CODE], or [CODE]DISABLED[/CODE]
[*][CODE]update-download-url[/CODE] — link shown in update notifications (default GitHub releases)
[*]bStats plugin ID and block-reason charts
[/LIST]

[HEADING=3]preset[/HEADING]
Merge a built-in server preset on load: [CODE]none[/CODE], [CODE]theryn[/CODE], [CODE]skyblock[/CODE], [CODE]vanilla-survival[/CODE], [CODE]hub-spawn[/CODE], [CODE]testing[/CODE].

[HEADING=2]Integrations[/HEADING]
All integrations are optional soft dependencies. ItemMagnet runs standalone on Paper with no extra plugins required.

[LIST]
[*][B]Lands[/B] — wilderness, owner, member, and flag-based modes
[*][B]WorldGuard[/B] — region whitelist/blacklist and item-pickup flag
[*][B]Towny[/B] — town plot protection
[*][B]GriefPrevention[/B] — claim respect
[*][B]Residence[/B] — residence claim and itempickup flags
[*][B]PlotSquared[/B] — plot membership checks
[*][B]SuperiorSkyblock2[/B] — island permission hook
[*][B]CMI[/B] — stat/rank unlock gates
[*][B]LuckPerms[/B] — LP_GROUP unlock type
[*][B]mcMMO[/B] — skill level unlock gates
[*][B]Quests[/B] — quest-complete tier unlocks
[*][B]PlaceholderAPI[/B] — charge, radius, tier, boost, filter placeholders
[/LIST]

[HEADING=2]FAQ[/HEADING]

[HEADING=3]Does the magnet pull items through walls?[/HEADING]
No. ItemMagnet uses stepped movement with line-of-sight checks. Items stop when a solid block blocks the path.

[HEADING=3]Does it work without Lands or WorldGuard?[/HEADING]
Yes. Both are optional. With integrations disabled, only permission checks apply.

[HEADING=3]How do I fuel the magnet?[/HEADING]
[LIST=1]
[*]Shift + right-click with redstone dust or block in either hand
[*]Auto-absorb — drop redstone nearby while holding a charged magnet
[/LIST]

[HEADING=3]How do I disable update notifications?[/HEADING]
Set [CODE]metrics.update-check: DISABLED[/CODE] in config, or remove [CODE]itemmagnet.updates[/CODE] from staff.
